\section{Implications and Related Work}
\label{sec:discussion}
\textbf{Optimizer interfaces as semantic interfaces.}  The results suggest a reporting rule for optimizer studies: a public claim should name the action being compared, the guard under which it applies, the optimizer layer that owns it, the placement at which it executes, the decision time, and the evidence a user can observe.  These fields do not expose proprietary cost formulas; they state whether two engines are being compared on the same query-processing phenomenon.  In heterogeneous analytics, where optimizers span local engines, adapters, remote sources, and service-side stages \cite{haas1997garlic,stonebraker2015bigdawg,calcite_docs,datafusion_docs,presto_docs}, that boundary is decisive: a speedup or plan difference may reflect local rewrite, source delegation, exchange placement, runtime adaptation, or a different evidence surface.

\textbf{Theory, testing, provenance, and benchmarks.}  Classical optimizers separate logical equivalence, physical alternatives, access paths, search, costing, and runtime adaptation \cite{selinger1979access,graefe1990volcano,graefe1993volcano,graefe1995cascades,chaudhuri1998overview,ioannidis1996query}.  OptSem-C does not reconstruct private search; it asks which distinctions are public enough to support cross-engine comparison.  SQL-equivalence systems decide whether expressions preserve results \cite{chu2017cosette,chu2018usemiring,wang2024qed,chandra1977optimal,aho1979equivalences}; OptSem-C studies a different relation in which result-equivalent queries can expose non-equivalent optimizer contracts.  Provenance work supplies the methodological precedent that support should be attached to the object being compared \cite{green2007provenance,buneman2001why,cheney2009provenance,green2010provenance}.  Exact-cardinality testing, JOB, adaptive processing, and learned-optimizer work similarly sharpened denominators when older metrics hid the behavior of interest \cite{leis2015good,leis2018looking,chaudhuri2009exact,markl2004progressive,kabra1998reoptimization,deshpande2007adaptive,avnur2000eddies,marcus2019neo,marcus2021bao,zhu2023lero}.  TPC-H, TPC-DS, SSB, LDBC, Wisconsin, AS3AP, LinkBench, ClickBench, CardBench, and LakeBench make workloads, labels, or evaluation denominators explicit \cite{tpcds,tpch,oneil2009ssb,erling2015ldbc,bitton1983wisconsin,turbyfill1989as3ap,armstrong2013linkbench,clickbench,chronis2024cardbench,deng2024lakebench}.  OptSemBench-C borrows that discipline but uses query probes over optimizer-decision feature interactions rather than a single latency score.

\textbf{Comparative-study protocol and scope.}  A future optimizer comparison can use OptSem-C as a front-end calibration step: state the public contract, the guard, and the semantic fields that make the behavior comparable.  If exact signatures agree, performance or plan-quality measurements have a clear denominator.  If a coarse projection collapses unequal signatures, the measurement can still be reported, but the portability claim must name the missing field.  The field lattice matters because the answer is neither ``all details'' nor ``any feature label'': in the no-variant semantic universe, every singleton field is unsafe, while two-field semantic vocabularies can already reach the exact-equivalence denominator.  The scope is deliberately public.  OptSem-C does not infer private cost models, prove engine-internal equivalence, or replace latency evaluation; it makes optimizer-portability claims stateable, falsifiable, and repairable at the semantic resolution exposed to users.

\section{Conclusion}
A coarse optimizer comparison can fabricate portability even when every individual system statement is public and accurate.  OptSem-C formalizes this failure as a projection kernel over guarded public optimizer contracts.  It defines the contract object, gives a finite state-join and projection semantics, generates covering query probes, grounds contracts in public evidence, executes the generated probes, and computes the semantic fields that repair observed false equivalences.

Keyword projection declares 2,284 equivalences, 254 of which exact contracts refute; operator-only projection declares 2,268 equivalences, 238 of which are false.  Placement repairs keyword collisions, layer repairs operator-only collisions, and layer plus placement repairs all 498 headline witnesses when fine-grained action-variant labels are excluded.  Over the no-variant semantic universe, every singleton field is unsafe, but two-field semantic vocabularies already reach the exact-equivalence denominator.  A comparison that preserves the public contract surface can still disagree on speed, cost, or plan quality; a comparison that does not preserve it has already changed the object being measured.  Coarse optimizer matrices are therefore incomplete rather than unusable.  OptSem-C gives a finite test for that incompleteness and a finite repair when the test fails.
