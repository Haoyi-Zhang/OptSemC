\section{Discussion and Limitations}
\label{sec:discussion}

\textbf{Optimizer interfaces as semantic interfaces.}  Public optimizer claims should name action, guard, layer, placement, decision time, and evidence surface.  These fields do not expose private cost formulas; they keep adapters, remote sources, and service stages from collapsing into one phenomenon, a boundary visible in federation~\cite{carey1995garlic} and polystores~\cite{stonebraker2015bigdawg}.

\textbf{What changes in an evaluation.}  Before ranking latency, cardinality error, or plan quality, a study can test whether compared systems share the same public contract relation.  If a table equates connector delegation with runtime filtering, the study should retain separating fields, narrow the claim, or report mixed phenomena.

\textbf{Protocol, scope, and refresh.}  JOB, TPC-H, and TPC-DS standardize inputs and metrics, not contract equality.  Every optimizer comparison is still a projection: choose a vocabulary, extract public contracts, validate the denominator, compute the kernel, and publish retained fields.  Counts are lower bounds over public evidence; later evidence becomes a source refresh.

\section{Related Work}
\label{sec:related}

OptSem-C sits between optimizer architecture, benchmarks, and DBMS testing.  System R, Volcano, Cascades, Calcite, and Orca explain access paths, rule search, adapter boundaries, and optimizer modularity \cite{selinger1979access,graefe1990volcano,graefe1995cascades,begoli2018calcite,soliman2014orca}; federation, polystores, and cloud engines explain placement and evidence surfaces \cite{carey1995garlic,stonebraker2015bigdawg,dageville2016snowflake,melnik2010dremel}.  Benchmarks such as TPC-H, TPC-DS, JOB, and CardBench standardize workloads or estimation stress \cite{tpch,tpcds,leis2015good,chronis2024cardbench}; equivalence and testing tools check SQL semantics or DBMS bugs \cite{chu2017cosette,wang2024qed,rigger2020sqlancer,rigger2020norec}.  OptSem-C audits the public contract denominator before those results are compared across systems.

\section{Conclusion}
OptSem-C shows that coarse optimizer labels can equate different public contracts, so denominator validity must precede performance ranking.
