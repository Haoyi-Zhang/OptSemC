\textbf{Pushdown is an operator family.} Predicate movement, aggregation delegation, join delegation, limit delegation, and Top-N delegation are separate contracts.  Keyword projection fails because the semantic object is the guarded action, not the word ``pushdown''; placement separates local rewrite from source delegation.  \textbf{Plan evidence is not isomorphic.} Logical plans, estimated physical plans, runtime profiles, and distributed stage graphs are different public evidence states.  OptSem-C represents this distinction through observability and decision-time fields, so a cloud-service stage graph and a local estimated plan are not collapsed into one comparison object.  \textbf{Decision time changes semantics.} Compile-time join selection, stage-boundary repartitioning, and runtime adaptive execution are not variants of the same behavior; removing decision time makes adaptive and static contracts appear portable even when the public adaptive contract is strictly stronger \cite{pavlo2017selfdriving,kraska2018learnedindex,krishnan2018sage,yang2022bao,dutt2019selectivity,naru2019}.  \textbf{Materialization and reuse are guarded.} CTE reuse, volatility, grouped aggregation, and explicit materialization controls decide whether an intermediate is folded, materialized, or blocks pushdown.  Thus operator and placement repair pushdown collisions; observability and decision time repair plan-evidence collisions; guard and variant repair reuse and materialization collisions.  The cases connect aggregate false-equivalence counts to mechanisms a reviewer can inspect.
